## 📦 模块设计

### 1. **主爬虫脚本** (`scraper.py`)

```python
scraper.py
├── scrape_tech_news()           # 爬虫函数
│   ├── for each RSS feed:
│   │   ├── feedparser.parse()  # 解析 RSS
│   │   ├── 提取文章信息
│   │   └── 聚合文章列表
│   └── return articles[]
│
├── send_to_weixin()             # 推送函数
│   ├── 构建消息载荷
│   ├── requests.post()          # 发送 HTTP 请求
│   └── return success/failure
│
└── main()                       # 主函数
    ├── 获取环境变量
    ├── 调用爬虫函数
    ├── 调用推送函数
    └── 返回执行结果
```

### 2. **配置管理** (`config.py`)

```python
config.py
├── 路径配置
│   ├── BASE_DIR
│   ├── DATA_DIR
│   └── LOGS_DIR
│
├── 日志配置
│   ├── LOG_LEVEL
│   └── LOG_FILE
│
├── 爬虫配置
│   ├── SCRAPER_TIMEOUT   (10s)
│   ├── SCRAPER_RETRIES   (3次)
│   └── SCRAPER_DELAY     (1s)
│
└── 新闻源配置
    └── NEWS_SOURCES[]
```

### 3. **爬虫模块** (`scrapers/`)

```
scrapers/
├── __init__.py           # 模块导出
├── base_scraper.py       # 基础爬虫类
│   ├── class BaseScraper
│   ├── def run()
│   ├── def parse()
│   └── def close()
└── news_scraper.py       # 新闻爬虫实现
    ├── class NewsScraper(BaseScraper)
    └── RSS 源特定逻辑
```

### 4. **快速开始工具** (`quickstart.py`)

```python
quickstart.py
├── print_section()       # 格式化输出
├── check_setup()         # 检查配置
├── print_next_steps()    # 显示后续步骤
├── print_troubleshooting() # 故障排查指南
└── main()               # 主函数
```

## 🔄 数据流

### 爬虫执行流程

```
START
  │
  ├─► 加载环境变量
  │
  ├─► 初始化日志系统
  │
  ├─► 定义 RSS 源列表
  │   ├─ HackerNews
  │   └─ TechCrunch
  │
  ├─► FOR EACH RSS源:
  │   ├─► 发起 HTTP GET 请求
  │   │
  │   ├─► feedparser 解析
  │   │   ├─ 检查返回码
  │   │   ├─ 解析 XML/JSON
  │   │   └─ 提取 entries[]
  │   │
  │   ├─► FOR EACH 文章 (取前3条):
  │   │   ├─► 提取字段
  │   │   │   ├─ title (限128字)
  │   │   │   ├─ link
  │   │   │   ├─ summary (限200字)
  │   │   │   └─ published_at
  │   │   │
  │   │   ├─► 创建 Article 对象
  │   │   │
  │   │   └─► 添加到文章列表
  │   │
  │   └─► 异常处理
  │       ├─ 日志记录
  │       └─ 继续下一个源
  │
  ├─► 数据聚合
  │   └─ 合并所有文章 (通常 6-8 篇)
  │
  ├─► 构建微信消息
  │   ├─ msgtype: "news"
  │   ├─ 文章列表 (最多5篇)
  │   └─ 转换为 JSON
  │
  ├─► 发送到企业微信
  │   ├─► POST 请求到 API
  │   ├─► 检查响应码
  │   ├─► 解析 JSON 响应
  │   │   ├─ errcode == 0 ✓ 成功
  │   │   └─ errcode != 0 ✗ 失败
  │   │
  │   └─► 重试逻辑
  │       ├─ 失败 → 重试 (最多3次)
  │       └─ 超时 → 重新连接
  │
  ├─► 记录结果
  │   └─ 成功/失败日志
  │
  └─► EXIT (0=success, 1=failure)
```

### 推送消息格式

```json
{
  "msgtype": "news",
  "news": {
    "articles": [
      {
        "title": "新闻标题 (最多128字符)",
        "description": "摘要内容 (最多200字符)",
        "url": "https://example.com/article"
      },
      ...
    ]
  }
}
```

## 🎯 关键工作流程

### 工作流 1: 本地开发测试

```
开发环境
  │
  ├─► pip install -r requirements.txt
  │
  ├─► export WEIXIN_WEBHOOK_KEY="xxx"
  │
  ├─► python scraper.py
  │   ├─ 爬取新闻
  │   ├─ 推送到微信
  │   └─ 显示结果
  │
  └─► 查看企业微信群
      ├─ 收到消息 ✓
      └─ 出现问题 ✗ 调试
```

### 工作流 2: 自动化部署

```
GitHub 仓库推送
  │
  ├─► .github/workflows/daily-scraper.yml
  │   ├─ on: schedule (09:00 UTC daily)
  │   ├─ on: workflow_dispatch (手动)
  │
  ├─► Ubuntu 环境准备
  │   ├─ Checkout 代码
  │   ├─ 安装 Python 3.11
  │   └─ 安装依赖
  │
  ├─► 执行 python scraper.py
  │   ├─ 从 Secret 注入 WEIXIN_WEBHOOK_KEY
  │   ├─ 爬取新闻
  │   ├─ 推送消息
  │   └─ 记录日志
  │
  ├─► 企业微信接收
  │   └─ 群组显示新闻卡片
  │
  └─► 监控与告警
      ├─ 成功 → 记录
      └─ 失败 → 发送告警
```

### 工作流 3: 扩展新闻源

```
需求: 添加新的 RSS 源 (Medium)
  │
  ├─► 修改 scraper.py
  │   ├─ RSS_FEEDS.append({
  │   │   'name': 'Medium',
  │   │   'url': 'https://...'
  │   │ })
  │   └─ 测试解析
  │
  ├─► 本地验证
  │   └─ python scraper.py (无误后提交)
  │
  ├─► 推送到 GitHub
```

## 📊 系统容量规划

| 指标 | 当前值 | 最大承载 | 优化方向 |
|------|--------|---------|----------|
| RSS 源数 | 2 | 10+ | 配置表 |
| 每源条目数 | 3 | 10 | 按需调整 |
| 并发请求 | 序列 | 支持并发 | asyncio |
| 存储方式 | 内存 | 数据库 | SQLite/PostgreSQL |
| 去重机制 | 无 | 哈希/DB | URL 去重 |
| 推送频率 | 1次/天 | 可调节 | 多次/小时 |

---

## 🚀 性能优化建议

1. **并发爬虫** - 使用 asyncio 并发获取多个 RSS 源
2. **缓存机制** - Redis/内存缓存去重
3. **数据库** - 存储历史文章避免重复
4. **增量更新** - 只推送新文章
5. **消息分页** - 超过限制时分批推送
6. **监控指标** - 记录爬虫性能数据

## 🔍 故障排查流程

```
问题发生
  │
  ├─► 检查 Actions 日志
  │   └─ 日志位置: GitHub Repo → Actions → 历史记录
  │
  ├─► 常见问题排查
  │   ├─ 无消息 → 检查 Webhook Key
  │   ├─ 连接超时 → 检查网络
  │   ├─ RSS 解析失败 → 检查 URL
  │   └─ 推送失败 → 检查微信 API 状态
  │
  ├─► 本地复现
  │   └─ export key; python scraper.py
  │
  └─► 查看日志文件
      └─ logs/ainews.log
```
