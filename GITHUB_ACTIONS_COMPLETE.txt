【步骤 1】配置 GitHub Secret

  1. 打开你的 GitHub 仓库
  2. Settings → Secrets and variables → Actions
  3. 点击 "New repository secret"
  4. Name: WEIXIN_WEBHOOK_KEY
  5. Value: 你的企业微信 Webhook Key (从 URL 中复制)
  6. Add secret

  ✅ Secret 配置完成


【步骤 2】推送代码到 GitHub

  git add .
  git commit -m "Add GitHub Actions workflows"
  git push origin main

  ✅ 代码已推送


【步骤 3】启用并测试工作流

  1. 进入 GitHub 仓库 → Actions 标签
  2. 应该看到 3 个工作流
  3. 点击 "Daily Tech News Scraper"
  4. 点击 "Run workflow"
  5. 等待完成
  6. 查看企业微信群，应该收到新闻推送

  ✅ 工作流测试完成

【Daily Scraper 工作流】⭐ 推荐

  执行步骤:
  1. 检出代码
  2. 安装 Python 3.11
  3. 安装依赖
  4. 验证依赖包
  5. 运行爬虫脚本
  6. 输出执行状态

  执行时间: 每天 UTC 9:00 (北京时间 17:00)
  手动触发: Actions → "Run workflow"
  预计耗时: 1-2 分钟


【Advanced Scraper 工作流】🔄 自动重试

  执行步骤:
  1. 第 1 次尝试
  2. 失败 → 第 2 次尝试
  3. 失败 → 第 3 次尝试
  4. 记录详细日志
  5. 发送完成通知

  执行时间: UTC 9:00 和 14:00 (备用)
  重试机制: 最多 3 次
  预计耗时: 1-5 分钟
  优势: 网络抖动时自动恢复


【Quality Check 工作流】✔️ 代码质量

  执行内容:
  
  质量检查:
    • 依赖完整性检查
    • Black 代码风格检查
    • isort 导入顺序检查
    • Flake8 代码规范检查
    • Pytest 单元测试
    • 模块导入测试
    • 脚本执行测试

  安全检查:
    • Bandit 安全扫描
    • 密钥泄露检查

  依赖检查:
    • Safety 漏洞扫描
    • pip-audit 版本检查

  触发条件: 每次 push 到 main/master/develop

问题: 工作流不运行
解决:
  1. 检查 Secret 是否配置 (Settings → Secrets)
  2. 检查 Actions 是否启用
  3. 查看工作流文件是否有语法错误
  4. 检查代码是否成功推送

问题: 收不到企业微信推送
解决:
  1. 检查 Secret 值是否正确
  2. 检查工作流日志中的错误
  3. 手动运行 python scraper.py 测试
  4. 检查 RSS 源是否可访问

问题: 代码质量检查失败
解决:
  1. 查看详细的检查报告
  2. 修复代码问题
  3. 重新推送代码
  4. 工作流会自动重新运行

  → 参考 GITHUB_ACTIONS_GUIDE.md 的故障排查部分
